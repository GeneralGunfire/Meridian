# Meridian Backend API

## Setup

```bash
npm install
npm run seed      # Generate dummy test data
npm run dev       # Start development server (localhost:3000)
```

## API Endpoints

### GET `/api/files`
Returns all available data files grouped by dataset.

**Response:**
```json
{
  "crime_stats": [
    {
      "week": "2026-w21",
      "date": "2026-05-26",
      "csv_url": "/api/download/crime_stats_2026-w21.csv",
      "xlsx_url": "/api/download/crime_stats_2026-w21.xlsx",
      "rows": 1247
    }
  ],
  "eskom": [...],
  "water": [...],
  "housing": [...]
}
```

### GET `/api/status`
Returns last scrape status for each dataset.

**Response:**
```json
{
  "crime_stats": {
    "success": true,
    "last_updated": "2026-05-26",
    "error": null
  },
  "eskom": {...},
  "water": {...},
  "housing": {...}
}
```

### GET `/api/download/:filename`
Downloads a specific CSV/XLSX file.

**Example:** `/api/download/crime_stats_2026-w21.csv`

## Data Structure

### Datasets

**Crime Stats**
```
Week, Year, Province, Crime_Type, Count
2026-w21, 2026, Gauteng, Homicide, 145
```

**Eskom**
```
Date, Stage, Province
2026-05-26, 2, National
```

**Water**
```
Year, Municipality, Water_Supply_Percent, Sanitation_Percent
2023, Johannesburg, 87.2, 83.1
```

**Housing**
```
Year, Total_Households_Millions, Formal_Dwellings_Percent, Electricity_Percent, Water_Percent
2023, 19.0, 58.6, 89.8, 87.0
```

## Scrapers

Located in `/src/scrapers/`:

- `crime-scraper.ts` — SAPS crime statistics
- `eskom-scraper.ts` — Eskom load shedding
- `water-scraper.ts` — Water supply statistics
- `housing-scraper.ts` — Housing data
- `scrape-all.ts` — Orchestrates all scrapers
- `utils.ts` — File saving, week calculation

**Run all scrapers:**
```bash
npm run scrape    # (after TypeScript build)
```

## GitHub Actions

Workflow file: `.github/workflows/scrape-data.yml`

**Triggers:**
- Weekly on Monday at 9:00 AM UTC
- Manual trigger via GitHub Actions UI

**What it does:**
1. Installs dependencies
2. Builds TypeScript scrapers
3. Runs all scrapers
4. Commits data files to repo
5. Pushes changes

## File Storage

All data stored in `/data/` directory:
```
data/
├── crime_stats_2026-w21.csv
├── eskom_2026-w21.csv
├── water_2026-w21.csv
├── housing_2026-w21.csv
└── status.json
```

Files are version controlled in Git. Each week's scrape creates new files (doesn't replace old ones).

## Development Notes

- Dummy data is generated by `npm run seed` for local testing
- All scrapers currently return dummy data (real scraping logic can be added)
- API automatically detects files in `/data/` and serves them
- Error handling is in place for missing files, invalid filenames, etc.
